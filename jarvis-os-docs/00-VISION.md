# Vision

## Product statement

JARVIS OS helps one person continue and complete computer work through natural language, voice, and contextual UI while preserving user agency. Its first excellent workflow is:

> “Continue my project.”

JARVIS identifies an explicitly registered project, checks repository and dependency health, proposes a plan, obtains any required approvals, opens the tools, starts services, verifies readiness, and reports actionable failures.

## Principles

- Local-first, cloud-optional.
- Useful while offline for core desktop and project workflows.
- Progressive autonomy: observe → suggest → preview → approve → execute → verify.
- Narrow specialists behind one orchestrator, not a society of agents talking indefinitely.
- Deterministic policy around probabilistic reasoning.
- Accessibility, keyboard operation, and visible system state from the first UI.
- No covert surveillance, emotion inference, credential collection, or lock-screen bypass.

## Initial user

A Windows developer with multiple local projects who uses VS Code, Git, Docker, terminals, browsers, PostgreSQL, and Redis.

## North-star outcome

Median time from a valid “continue project” request to a verified ready workspace is under 90 seconds, with zero unapproved high-risk side effects.

## Non-goals for v1

- General autonomous computer use on unknown interfaces.
- Unlocking Windows or bypassing OS security boundaries.
- Continuous camera recording.
- Autonomous email sending, purchases, publishing, production deployment, or remote pushes.
- Self-modifying permissions or prompts.
- Claims about health, emotion, identity, or intent from camera imagery.

## Success measures

- Task success and verified-success rates.
- Approval precision: risky actions never execute without the right approval.
- Recovery rate after interruption or restart.
- Time saved per repeated workflow.
- User cancellation latency and audit comprehensibility.
- Local resource use at idle and during a task.
