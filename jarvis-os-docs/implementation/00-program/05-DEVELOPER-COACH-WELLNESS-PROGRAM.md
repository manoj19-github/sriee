# Developer Coach and Wellness Delivery Program

Source: [developer coach and wellness companion](../../42-DEVELOPER_COACH_WELLNESS.md).

## Dependencies

- Authenticated task/event lifecycle and cancellation.
- Policy, approval, audit and privacy centers.
- Registered projects and scoped developer adapters.
- Consented memory with correction/deletion.
- Notification quiet hours and proactive-prompt controls.
- Provider-neutral connector interface with freshness and revocation.

## Slice 1 — Personality and developer profile

1. Store a versioned tone profile, work/casual mode, humor level and excluded topics.
2. Store approved skills, learning goals, projects, preferred tools and working hours.
3. Require uncertainty/source labels and “I don't know” behavior.
4. Add correction, reset and export controls.

Gate: hallucinated progress/history, inappropriate humor, stale data and profile-reset tests pass.

## Slice 2 — Developer briefing and conversation

1. Compose freshness-tagged morning/evening developer briefings.
2. Add bounded proactive greeting and workspace-start offers.
3. Add weekend and relaxation modes.
4. Enforce category cooldown, daily limit, quiet/private mode and one-action mute.

Gate: duplicate trigger, quiet hour, locked screen, failed connector, stale source and stop/mute tests pass.

## Slice 3 — Workflow companion

1. Consume typed build/test/container/Git/system observations.
2. Detect repeated, bounded friction patterns without reading new content.
3. Ask before inspecting additional logs/files/processes.
4. Route approved diagnosis and fixes through the standard task/executor path.
5. Summarize code-review evidence and verified outcomes.

Gate: secret redaction, false correlation, stale logs, no-consent inspection, dirty tree, destructive action and unsupported-cause tests pass.

## Slice 4 — Learning and AI mentor

1. Create curricula for chosen subjects and course sources.
2. Track trusted or user-confirmed progress.
3. Deliver lessons, quizzes and original coding challenges.
4. Explain mistakes and recommend the next topic with rationale.
5. Add adaptive reminders without shame or engagement pressure.

Gate: false completion, copied protected course content, incorrect scoring, inaccessible challenge, reminder fatigue and delete/reset tests pass.

## Slice 5 — Projects, achievements and career

1. Convert goals into roadmaps, milestones, sprint candidates and dependencies.
2. Link progress only to durable tasks, receipts and explicit confirmation.
3. Celebrate verified milestones and compose evidence-based motivation.
4. Add resume/LinkedIn review and mock coding/system-design/behavioral interviews.
5. Preview and approve every external profile/application mutation.

Gate: fabricated achievement, private-data leak, biased interview feedback, salary certainty, external-write approval and rollback tests pass.

## Slice 6 — Wellness connectors and records

1. Add manual entry before third-party connectors.
2. Register provider/device, data type, units, scope, freshness, retention and egress.
3. Store sensitive wellness data separately from general companion memory.
4. Display source values and device-supplied flags without diagnostic reinterpretation.
5. Add correction, export, disconnect and verified deletion.

Gate: unit mismatch, stale/conflicting source, revoked connector, cross-user leak, cloud-egress denial, deletion and lock-screen privacy tests pass.

## Slice 7 — Care reminders and check-ins

1. Add hydration, movement, meal, eye-rest, sleep and user-configured medication reminders.
2. Keep medication text/schedule exact and prohibit dose/missed-dose advice.
3. Add user-authored goals and adjustable accessibility/ability settings.
4. Add self-reported emotional check-ins and sensitive mood journal.
5. Escalate urgent/severe symptom language to appropriate professional/emergency help without diagnosis.

Gate: duplicate/DST reminder, missed-dose request, abnormal-reading request, emergency language, repeated probing, dismissal, quiet hour and deletion tests pass.

## Slice 8 — Resource and lifestyle assistance

1. Observe typed CPU/RAM/Docker/battery signals.
2. Propose owned-resource cleanup or pause actions.
3. Require approval before stopping containers/processes or changing system settings.
4. Offer music, rest or tomorrow-plan options at workday end.

Gate: unrelated-process protection, unsaved work, stale telemetry, approval, cancellation and verified postcondition tests pass.

## Release gate

No function ships as complete until functional, privacy, security, accessibility and failure-path evidence exists. Wellness claims receive product/legal review against the deployment jurisdiction. The release UI states that Sriee provides general information and organization—not medical diagnosis or treatment.
